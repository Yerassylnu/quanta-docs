# Search, System Map, Analytics

## Search (Command Palette)

Search открывается через:
- `Ctrl/Cmd + K`,
- Sidebar -> `Search`,
- верхнее меню -> `Selection` -> `Search`.

Поиск работает по:
- имени файла,
- содержимому файла (`.md`, `.txt`).

Особенности:
- используется fuzzy-поиск (находит даже при небольших ошибках в запросе),
- есть подсветка совпадений,
- для контента показывается сниппет вокруг найденного фрагмента,
- индекс поиска обновляется после изменений Workspace,
- при вводе в редакторе индекс текущего файла обновляется в реальном времени.

## Поведение при выборе результата поиска

При открытии результата:
- открывается соответствующий файл,
- приложение переключается в `Editor`,
- найденный запрос подсвечивается в редакторе,
- редактор прокручивается к первому подсвеченному совпадению.

## System Map

`System Map` — это граф файловой структуры Workspace.

Что отображается:
- узлы папок и файлов,
- иерархические связи (родитель -> ребенок),
- wiki-ссылки между файлами (`[[FileName]]`) как отдельные пунктирные связи.

Управление:
- клик по папке: раскрыть/свернуть узел,
- клик по файлу: открыть в Editor,
- поддерживаются zoom/pan через контролы React Flow.

Wiki-link формат:
- В тексте файла пишите `[[ИмяФайла]]` или `[[ИмяБезРасширения]]`.
- При наличии целевого файла появится связь в карте.

## Analytics

`Analytics` считывает трекеры из markdown/text файлов и отображает прогресс.

Формат трекера:
- `$label = current / target`
- Примеры:
- `$books = 3 / 12`
- `$sales = 42 / 100`

Поведение:
- если трекер с label встречается один раз, показывается editable-карточка,
- если label встречается в нескольких файлах, показывается агрегированная сумма,
- клик по карточке одиночного трекера открывает исходный файл,
- значение можно редактировать прямо в карточке (`Enter` для подтверждения).

Обновление:
- кнопка refresh в Analytics обновляет данные,
- после изменения значения трекера данные пересчитываются автоматически.
